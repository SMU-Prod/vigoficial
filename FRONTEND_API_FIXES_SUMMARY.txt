IMPLEMENTATION DETAILS & TESTING
================================================================================

CSRF (FE-02):
- Pattern: Double-submit cookie
- Token size: 256 bits (random)
- Storage: httpOnly=false in cookie (accessible to JS)
- Header: x-csrf-token (sent by JavaScript in form submission)
- Comparison: crypto.timingSafeEqual() prevents timing attacks
- Exemptions: /api/auth/login, /api/auth/register, /api/webhooks/*

GPS Webhook (FE-03):
- Algorithm: HMAC-SHA256
- Input: Raw request body (JSON string)
- Format: Hex digest in x-gps-signature header
- Verification: Constant-time comparison
- Test: openssl dgst -sha256 -mac HMAC -macopt key:SECRET

Token Refresh (FE-05):
- Trigger: HTTP 401 response
- Action: POST /api/auth/refresh
- Retry: Original request after token updated
- Limit: Max 1 retry per request (prevents loops)
- Reset: On URL change or final error

SECURITY IMPROVEMENTS
================================================================================

1. Timing Attack Prevention
   - CSRF token comparison: crypto.timingSafeEqual()
   - GPS webhook signature: crypto.timingSafeEqual()

2. Type Safety
   - noImplicitAny enabled (prevents accidental type errors)
   - Proper typing for API responses

3. CSRF Protection
   - Double-submit cookie pattern
   - Prevents cross-site request forgery

4. Webhook Security
   - HMAC-SHA256 signature verification
   - Replaces weak shared-secret string comparison

5. Session Management
   - Automatic token refresh on expiration
   - Transparent to user (no re-authentication prompt)
